docker内拨号

（头条是主机拨号）

如果之前有服务正在运行则停掉容器
docker kill `docker ps -a --format "table {{.ID}}\t{{.Names}}" |grep -E 'ksp2p_|pcdn_|cp2p_'|awk '{print $1}'`

拨号镜像地址

registry.cn-hangzhou.aliyuncs.com/pcdn-images/pppoe:v5.9

拨号脚本

curl -sSL https://automateplus.oss-cn-hangzhou.aliyuncs.com/docker_pppoe_multieth.sh|bash -s -- -c 拨号镜像版本

创建拨号虚拟网卡脚本

curl -sSL https://automateplus.oss-cn-hangzhou.aliyuncs.com/host_pppoe_vlan.sh|sh

查看网卡，有state up的，并且有ip的网卡是主机网卡（内网ip），有state up，并且没有ip的是拨号网卡

拨号步骤

  1. 创建拨号文件夹
    mkdir /opt/pppoe
  2. 创建拨号配置文件
    /opt/pppoe/pppoe_user.txt
    文件格式：
    vlanid 拨号账户 账户密码 拨号网卡名 带宽大小（M）
    例：
    novlan 77561213412 00000000 ens3 100
    novlan 77539581907 00000000 ens3 100
    novlan 77515478098 00000000 ens3 100
    novlan 77531248304 00000000 ens3 100
    注意：第一列为vlanid，如果线路未设置vlan，则值为“novlan”
  3. 检验拨号是否成功

拨号失败的解决方案
手动恢复拨号

  1. 进入拨号容器
    docker exec -it pppoe_1 bash
  2. 启动ppp0网卡，eth0网卡
    ifup ppp0
    ifup eth0
    (ifup eth0 启动eth0网卡)
    (ifdown eth0 关闭eth0网卡)

然后再检验拨号是否成功

分盘部署

curl -sSL https://automateplus.oss-accelerate.aliyuncs.com/container-partition.sh | bash -s -- -c {pcdn/vdp/cp2p/p2p/ksp2p} $num $version

看看部署的容器的类型、数量、版本、然后依次填入

分盘部署完了之后再次使用create pppoe函数，直接复制到shell里，然后启动，也可以在本地服务器直接使用hyp执行脚本

检验是否拨号成功
查看任意一个容器的NetworkMode，如果为container，则该容器使用的是容器的拨号网络，表示OK
如果为host，表示容器使用的网络仍然为主机网络，则需要将网络格式改为拨号网络
运行pppoe.sh中的create_pppoe即可，然后再次查看任意一个容器的网络模式

显示所有容器
docker ps -a

查看任意一个容器的网络模式（以pcdn为例）
docker inspect pcdn_1| grep Net
